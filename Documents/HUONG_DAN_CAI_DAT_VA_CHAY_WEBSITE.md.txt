# HƯỚNG DẪN CÀI ĐẶT VÀ CHẠY WEBSITE VAN ANH PERFUME

## 1. Giới thiệu dự án

**VAN ANH PERFUME** là website quản lý và bán hàng nước hoa trực tuyến.

Website được xây dựng nhằm hỗ trợ khách hàng xem sản phẩm, tìm kiếm sản phẩm, xem chi tiết sản phẩm, thêm sản phẩm vào giỏ hàng, đặt hàng và theo dõi đơn hàng. Đồng thời, hệ thống cung cấp khu vực quản trị để Admin quản lý sản phẩm, danh mục, thương hiệu, đơn hàng, người dùng và các nội dung liên quan.

## 2. Chức năng chính của hệ thống

### 2.1. Phía khách hàng

- Xem trang chủ.
- Xem danh sách sản phẩm.
- Tìm kiếm sản phẩm.
- Xem chi tiết sản phẩm.
- Thêm sản phẩm vào giỏ hàng.
- Cập nhật số lượng sản phẩm trong giỏ hàng.
- Đặt hàng.
- Đăng ký tài khoản.
- Đăng nhập tài khoản.
- Quản lý thông tin cá nhân.
- Xem lịch sử đơn hàng.

### 2.2. Phía quản trị viên

- Đăng nhập trang quản trị.
- Quản lý sản phẩm.
- Quản lý danh mục.
- Quản lý thương hiệu.
- Quản lý biến thể sản phẩm.
- Quản lý hình ảnh sản phẩm.
- Quản lý tồn kho.
- Quản lý đơn hàng.
- Quản lý người dùng.
- Quản lý bài viết hoặc tin tức nếu hệ thống có chức năng này.
- Quản lý banner hoặc khuyến mãi nếu hệ thống có chức năng này.

# BƯỚC 2: CẤU HÌNH CONNECTION STRING

## 2.1. Vị trí file cấu hình
Đường dẫn tương đối:
VanAnhPerfume/VanAnhPerfume

Các file cấu hình thường dùng:
VanAnhPerfume/VanAnhPerfume/appsettings.json
VanAnhPerfume/VanAnhPerfume/appsettings.Development.json
VanAnhPerfume/VanAnhPerfume/appsettings.example.json

## 2.2. Cách cấu hình an toàn

Nếu có file: VanAnhPerfume/VanAnhPerfume/appsettings.example.json

Hãy copy file này thành: VanAnhPerfume/VanAnhPerfume/appsettings.Development.json

Sau đó sửa connection string trong `appsettings.Development.json`.


## 2.3. Connection string khi dùng SQL Server Express
Nếu SQL Server của bạn là `SQLEXPRESS`, dùng mẫu sau:
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.\\SQLEXPRESS;Database=VanAnhPerfume;Trusted_Connection=True;TrustServerCertificate=True;"
  }
}

## 2.4. Connection string khi dùng SQL Server mặc định
Nếu Server name là `localhost`, dùng mẫu sau:
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=VanAnhPerfume;Trusted_Connection=True;TrustServerCertificate=True;"
  }
}

Hoặc:
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.;Database=VanAnhPerfume;Trusted_Connection=True;TrustServerCertificate=True;"
  }
}

## 2.5. Connection string khi dùng tài khoản SQL Server
Nếu dùng tài khoản `sa` hoặc tài khoản SQL Server riêng, dùng mẫu sau:
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=VanAnhPerfume;User Id=sa;Password=YOUR_PASSWORD;TrustServerCertificate=True;"
  }
}


## 2.6. Kiểm tra tên connection string trong Program.cs

Mở file: VanAnhPerfume/VanAnhPerfume/Program.cs

Tìm dòng cấu hình database, ví dụ:
builder.Configuration.GetConnectionString("DefaultConnection")

Nếu code gọi: DefaultConnection
thì trong `appsettings.json` hoặc `appsettings.Development.json` cũng phải có:
{
  "ConnectionStrings": {
    "DefaultConnection": "..."
  }
}
Nếu tên không khớp, website sẽ không kết nối được database.

## 2.7. Xem hướng dẫn connection string chi tiết

Nếu cần xem riêng phần hướng dẫn cấu hình chuỗi kết nối, mở file: Documents/HuongDanCauHinhConnectionString.txt

# BƯỚC 2.8: CẤU HÌNH GỬI EMAIL BẰNG GMAIL SMTP

Website có sử dụng SMTP để gửi email, ví dụ: gửi email xác nhận, thông báo đơn hàng hoặc thông báo hệ thống.

Dự án sử dụng Gmail SMTP với cấu hình mẫu như sau:
"Smtp": {
  "Host": "smtp.gmail.com",
  "Port": "587",
  "Username": "your-email@example.com",
  "Password": "YOUR_EMAIL_APP_PASSWORD",
  "From": "your-email@example.com"
}

## 2.8.1. Ý nghĩa từng trường cấu hình SMTP

| Trường | Ý nghĩa | Cách điền |
| `Host` | Máy chủ SMTP dùng để gửi email | Giữ nguyên `smtp.gmail.com` |
| `Port` | Cổng gửi email SMTP | Giữ nguyên `587` |
| `Username` | Tài khoản Gmail dùng để gửi email | Điền đầy đủ địa chỉ Gmail, ví dụ `example@gmail.com` |
| `Password` | Mật khẩu ứng dụng Gmail | Điền Gmail App Password, không dùng mật khẩu Gmail thường |
| `From` | Email hiển thị là người gửi | Thường điền giống `Username` |

## 2.8.2. Không dùng mật khẩu Gmail thường

Không điền mật khẩu đăng nhập Gmail thông thường vào file cấu hình.

Để gửi email qua Gmail SMTP, cần tạo **App Password**. Đây là mật khẩu ứng dụng riêng dùng cho website hoặc phần mềm gửi email.

## 2.8.3. Bật xác minh 2 bước cho tài khoản Gmail

Trước khi tạo App Password, tài khoản Gmail cần bật xác minh 2 bước.

Các bước thực hiện:

1. Đăng nhập tài khoản Google.
2. Vào trang quản lý tài khoản Google.
3. Chọn mục **Security** hoặc **Bảo mật**.
4. Chọn **2-Step Verification** hoặc **Xác minh 2 bước**.
5. Làm theo hướng dẫn để bật xác minh 2 bước.

Nếu chưa bật xác minh 2 bước, tài khoản thường sẽ không tạo được App Password.

## 2.8.4. Tạo Gmail App Password

Sau khi bật xác minh 2 bước:

1. Vào tài khoản Google.
2. Chọn **Security** hoặc **Bảo mật**.
3. Tìm mục **App passwords** hoặc **Mật khẩu ứng dụng**.
4. Tạo mật khẩu ứng dụng mới.
5. Đặt tên dễ nhớ, ví dụ: VanAnhPerfume SMTP
```
Google sẽ sinh ra một mã gồm 16 ký tự.

Ví dụ dạng hiển thị: abcd efgh ijkl mnop

Khi điền vào file cấu hình, có thể nhập liền không có dấu cách: abcdefghijklmnop

Lưu ý: App Password thường chỉ hiển thị một lần. Nếu quên mã này, cần xóa mã cũ và tạo mã mới.

## 2.8.5. Điền thông tin vào appsettings.Development.json
Người chạy project cần copy file:

VanAnhPerfume/VanAnhPerfume/appsettings.example.json

thành:

VanAnhPerfume/VanAnhPerfume/appsettings.Development.json

Sau đó sửa phần `Smtp` trong `appsettings.Development.json`.

Ví dụ:

```json
"Smtp": {
  "Host": "smtp.gmail.com",
  "Port": "587",
  "Username": "example@gmail.com",
  "Password": "abcdefghijklmnop",
  "From": "example@gmail.com"
}
```

Trong đó:

- `Username`: là Gmail dùng để gửi email.
- `Password`: là Gmail App Password vừa tạo.
- `From`: thường giống với `Username`.

## 2.8.6. Lỗi thường gặp khi gửi email Gmail SMTP

### Lỗi đăng nhập SMTP thất bại

Nguyên nhân thường gặp:

- Chưa bật xác minh 2 bước cho tài khoản Gmail.
- Chưa tạo App Password.
- Nhập nhầm mật khẩu Gmail thường thay vì App Password.
- Nhập sai địa chỉ Gmail trong `Username`.
- App Password bị thu hồi hoặc đã đổi mật khẩu Google.

Cách xử lý:

- Kiểm tra lại tài khoản Gmail.
- Tạo App Password mới.
- Cập nhật lại `Password` trong `appsettings.Development.json`.

### Không thấy mục App Password

Có thể do:

- Tài khoản chưa bật xác minh 2 bước.
- Tài khoản thuộc trường học hoặc công ty bị quản trị viên chặn.
- Tài khoản đang bật chế độ bảo vệ nâng cao.
- Tài khoản không đủ điều kiện tạo App Password.

### Đổi mật khẩu Google xong website không gửi được email

Khi đổi mật khẩu Google, App Password có thể bị thu hồi. Khi đó cần tạo App Password mới và cập nhật lại trong `appsettings.Development.json`.

- Email xác nhận hoặc email thông báo không được gửi đi.
- Log hiển thị lỗi xác thực SMTP.
- Gmail từ chối đăng nhập.

Cách xử lý:

1. Kiểm tra trong `appsettings.Development.json` đã có phần cấu hình `Smtp` chưa:

"Smtp": {
  "Host": "smtp.gmail.com",
  "Port": "587",
  "Username": "example@gmail.com",
  "Password": "abcdefghijklmnop",
  "From": "example@gmail.com"
}

2. Kiểm tra `Username` có phải Gmail đầy đủ không.

Ví dụ đúng: example@gmail.com

Ví dụ sai: example

3. Kiểm tra `Password` có phải Gmail App Password không.

Không dùng mật khẩu đăng nhập Gmail thông thường.

4. Kiểm tra tài khoản Gmail đã bật xác minh 2 bước chưa.

5. Nếu App Password cũ không dùng được, hãy xóa App Password cũ và tạo App Password mới.

6. Kiểm tra mạng internet và thử gửi lại email.
